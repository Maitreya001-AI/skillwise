# ablation — the standing counterexample audit (design)

**Claim under audit.** THEORY §2's exhaustiveness: every load-bearing fragment of an effective skill is attributable to one of the four loop sites — Σ (generation), Π (action), φ (verification), γ (scheduling). §2 volunteers this as a falsifiable prediction; this protocol is the audit made standing, so the theory's death stays orderable at all times.

## Protocol

1. **Pick a demonstrably effective skill** — one whose `delta_exist` cleared the gate beyond the band on a fit, adequately-powered set (an uncertified skill has no load-bearing fragments to attribute).
2. **Segment** it into semantically coherent fragments: a rule, a table row, a `scripts/` primitive, a description clause.
3. **Ablate one fragment at a time**; re-run the held-out set under the gate's protocol (fresh contexts, per-task medians, the §8-1 band from 3 reference re-runs, blind decorrelated judges).
4. A fragment is **load-bearing** iff its removal drops the aggregate beyond the noise band.
5. **Attribute** each load-bearing fragment to Σ / Π / φ / γ, recording the site and the one sentence that justifies the attribution.

## Outcome handling

- **All load-bearing fragments attributable** → the prediction survives this instance. It is never *proven* — §11-2: the audit only fails to falsify.
- **A load-bearing, unattributable residue** → **a theory counterexample.** Do not shoehorn it into a site. Report it against THEORY §11-1: the likely reading is that the intervention surface (§0) has grown a parameter position the four-atom read misses (e.g. skills that set sampling parameters, route models, or spawn sub-agent topologies), and the tuple must be re-derived.
