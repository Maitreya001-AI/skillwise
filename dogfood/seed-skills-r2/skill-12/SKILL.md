# Quarterly OKRs (org format)

## Format facts

- 2–4 objectives per team; 2–4 key results per objective; the whole set fits one page.
- Objectives are qualitative and inspiring; key results are the measurable claims underneath.
- Each key result names its metric source (the dashboard or table it will be read from).
- Committed vs aspirational: mark each objective `[C]` or `[A]`; the org expects roughly 70% attainment on aspirational sets.
- OKRs live in `planning/okrs/<year>-Q<n>.md`, one file per team.

## Quality

Make sure objectives are ambitious yet achievable, and genuinely inspiring rather than a task list in disguise. Key results should be truly measurable and outcome-oriented — avoid activity metrics where an outcome metric exists. The set as a whole should tell a coherent story about the quarter and feel right to the team that owns it. Aim for language that motivates.

## Never

- Never write a key result whose metric source doesn't exist yet.
- Never carry an objective over two quarters unchanged without an explicit "why still".
