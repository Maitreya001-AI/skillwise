# Weekly status report

The team's report has an expected shape and audience (stakeholders skim it; managers read the risks). Produce it as follows.

## Procedure

- **Step 1.** Collect this week's updates from the tracker and the merged PRs.
- **Step 2.** Group the updates by project, in the tracker's project order.
- **Step 3.** Draft the summary paragraph from the grouped updates.
- **Step 4.** After the summary is settled, write the risks section.
- **Step 5.** Once both sections read well, proceed to formatting: headings, then bullet polish, then send.

Follow the sequence as given — each step builds on the previous one, and skipping ahead produces drafts that have to be redone.

## What correct looks like

Every active project appears; each risk has an owner and a next review date; the summary is five sentences or fewer; stale items (no movement in 2+ weeks) are marked as such rather than silently dropped.
