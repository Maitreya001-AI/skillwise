# README polishing protocol

Small documentation edits deserve the same rigor as code. Every README change, however small, satisfies all of the following (they are requirements, not steps — meet them in any order):

## Requirements

- **Three alternatives**: produce three genuinely different drafts of the changed passage, not one draft and two paraphrases.
- **Scoring matrix**: score every draft on all five dimensions — clarity, concision, tone-fit, scannability, momentum — on a 1–10 scale, with a one-line justification per cell (15 cells total).
- **Cooling pass**: after selecting the winner, set it aside and re-read the entire README top to bottom, then re-read the winner in place.
- **Link audit**: resolve every link in the file — not just the edited section — twice, once before and once after the edit.
- **Rationale memo**: write a short memo (why this wording, what was rejected and why) and keep it alongside the PR.
- **Full spell/grammar pass** over the whole file, even for a one-line change.

## What correct looks like

The final README reads cleanly end to end, the chosen draft beats the alternatives on the matrix, all links resolve, and the memo exists.
