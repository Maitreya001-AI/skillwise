- Resource labels (the identifier after the type, e.g. `resource "aws_s3_bucket" "this"`) must not restate the resource type or provider — use `this` for the singleton resource in a module, or a role-based noun (`primary`, `replica`, `logs`) when multiple instances of the same type exist in one file.
- Actual cloud-facing names (the `name`/`bucket`/`Name` tag argument) must follow `<org>-<env>-<system>-<component>[-<qualifier>]`, all lowercase, hyphen-separated, no underscores, no embedded account IDs or regions unless the provider enforces global uniqueness and requires it.
- Every name-bearing argument must be built from `var.` / `local.` inputs via a shared naming module or `local.name_prefix`, never a hardcoded literal string, so environment and org values can't drift between resources.
- `count`/`for_each` keys must be stable, human-meaningful strings (map keys or a named list), never bare numeric indices, so resource addresses survive reordering without forcing replacement.
- Module names (the label in `module "..." {}`) must describe the module's purpose in the calling stack (`network`, `eks_cluster`), never the module source repo name or a version-suffixed name.
- File and directory names for reusable modules must mirror the module's resource domain (`modules/rds-postgres/`), and root-level stacks must be named after the environment/account they deploy (`live/prod-us-east-1/`), not generic terms like `main` or `infra`.
- Variable and output names must be prefixed to indicate their subject when a module exposes more than one resource of the same kind (`db_primary_endpoint`, `db_replica_endpoint`), never ambiguous singular names (`endpoint`) in a multi-resource module.
- Abbreviations are restricted to the org's approved list (documented once, e.g. `vpc`, `sg`, `iam`); any other abbreviation must be spelled out in full, and the approved list itself must not be extended ad hoc inside a PR.
- When importing existing resources, the Terraform label and `name` argument must be updated to match this grammar as part of the import PR — the pre-existing cloud name may stay if immutable, but the Terraform-side identifier must not perpetuate legacy naming schemes.
- Tags/labels carrying identity metadata (`Name`, `env`, `system`, `owner`) must be generated from the same `local.tags` map used for naming, never set independently per resource, so name and tags cannot diverge.
